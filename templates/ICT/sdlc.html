System devlopment Life cylcle

A system is a collection of components that interact together to achieve a specific task

A system converts data into information

Manual system

  • Errors are minimal which data processed by a computer program
  • Since there data is inserted manually processed there is room for more error
  • Processing of data is less efficien
  • Requires a large space for storage of data
  • Data is open to lots of threats

Some of the advantages of using a computer based syatem are

  • Errors are minimal which data processed by a computer program
  • Data can be processed more efficiently
  • Large amount of data can be stored in a small physical surface such as a database
  • Security can be ensured with backups and the use of passwords

Stages of system development life cycle

  • Identification of requirements
  • Designing the solution
  • Coding of the solution
  • Testing and debugging
  • Deployment of the system
  • Maintenance of the system

Some ways of identifying requirements

  • Observing
  • Interviews
  • Questioners
  • Document sample collection
  • Prototyping

Key points when we design a software

  • Identification of software and software architecture
  • User interface
  • Identification of main hardware system and its components
  • Identification of dependency of each sub system
  • Deciding on the hardware and software to run in the computer
  • Designing infrastructure for the software database and user interface
  • Planning of tests

The next step is to code the system using a suitable computer language

The main objective of testing and debugging is minimize the error in the above steps

There are few phase in testing

  • Unit testing
  • Integration testing
  • System testing
  • Acceptance testing

Each unit in the system is tested input are testing in accordance with the output this is called unit testing

Units that are already tested and debugged are integrated to form the complete system and the integrated system is tested this is called integration testing

The complete system is tested for it is output for the provided input this is called system testing

Acceptance of user is tested at last and this is known as acceptance testing

There are few different types of deployment

  • Direct deployment
  • Parallel deployment
  • Pilot deployment
  • Phased deployment

Direct deployment means the existing system is totally terminated and then the new system is installed

Parallel deployment is when the existing system and the new system are use parallel

Pilot deployment is about using the newly developed system in a selected area

In phased deployment the new system is deployed in stages success of one stage leads to the next and completing the new system

A system once installed and in use for a certain period needs to be maintained well with system maintenance the following may be required

  • Adopting new technology into the system to improve the efficiency
  • Issues that surfaced with the use of the system which were identified during the SDLC can be solved
  • Developing the new system to suit the new requirements

There are several models for SDLC each model facilitates the system in different ways

Some examples for theses models are

  • Waterfall model
  • Iterative incremental model
  • Prototyping model
  • Spiral model

The waterfall model consist of several stages of linear development

To develop system using the waterfall model

  • First identify requirements
  • Finish one stage before the next
  • The result of he developed system is found at the final stage

The result of he developed system is found at the final stage

The iterative incremental model allows software developers to take advantage of what was learned during the development and use it on the system

In this model a system is developed through repeated cycles and in smaller portions at a time
